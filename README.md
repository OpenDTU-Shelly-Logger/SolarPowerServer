# Solar & Power Monitor (Backend)

This project is a robust Python-based backend system designed to monitor, record, and process energy data from your home. It integrates with **OpenDTU** (for solar inverters) and **Shelly 3EM** (for grid power monitoring) to provide real-time insights and long-term historical analysis of your energy usage and production.

> **Note**: This is the **backend data collector** part of a larger system. It collects, processes, and stores data, then sends it to a separate API/Frontend server (hosted in a different repository) for visualization.

## 🌟 Key Features

The system operates in two main modes:

### 1. ⚡ Live Monitoring (Real-Time)

- **High Frequency**: Polls devices every **10 seconds**.
- **Data Points**:
  - Live Solar Power generation (from OpenDTU).
  - Live Grid Power consumption/export (from Shelly 3EM).
  - Inverter Status (Temperature, Yield).
- **purpose**: Provides immediate data for live dashboards to see what is happening right now.

### 2. 📊 Historical Analysis (Daily Batches)

- **Intelligent Aggregation**: The 10-second raw data is processed and compressed into efficient **10-minute batches**.
- **Daily Metrics**:
  - **Solar Produced**: Total energy generated by your panels.
  - **Exported to Grid**: Solar energy you didn't use.
  - **Bought from Grid**: Energy imported when solar wasn't enough.
  - **Self-Consumption**: Amount of solar energy used directly by your home.
  - **Peak Statistics**: Tracks the daily peak solar wattage and highest inverter temperature with timestamps.
- **Storage**: All historical data is persistently stored in a local SQLite database (`solar_data.db`) for long-term retention.

## 🛠 Tech Stack

- **Python 3**: Core language.
- **SQLite**: Local database for historical records.
- **Pandas**: Used for resampling high-frequency data into 10-minute intervals and calculating energy integrals.
- **OpenDTU**: Interface for Hoytmitles/generic inverters.
- **Shelly 3EM**: Smart energy meter integration.

## ⚙️ Configuration

The project is configured via a `.env` file. A sample is provided as `.env.example`.

### Environment Variables

| Variable           | Description                                                               |
| ------------------ | ------------------------------------------------------------------------- |
| `DOMAIN`           | The domain/IP of the separate API/Frontend server where data is uploaded. |
| `UPLOAD_API_KEY`   | Secure key for authenticating with the upload API.                        |
| `OPENDTU_IP`       | Local IP address of your OpenDTU device (e.g., `192.168.1.150`).          |
| `SHELLY_IP`        | Local IP address of your Shelly 3EM device (e.g., `192.168.1.25`).        |
| `DATABASE_PATH`    | Path to the local SQLite database (default: `./data/solar_data.db`).      |
| `BUFFER_FILE_PATH` | Path for temporary data buffering (default: `./data/buffer.json`).        |
| `LOG_FILE`         | Path where logs are written (default: `./data/output.log`).               |

## 🚀 Installation & Setup

1. **Clone the repository**

   ```bash
   git clone <repository-url>
   cd SolarPowerUpload
   ```

2. **Set up a virtual environment (Recommended)**

   ```bash
   # Windows
   python -m venv .venv
   .venv\Scripts\activate

   # Linux/Mac
   python3 -m venv .venv
   source .venv/bin/activate
   ```

3. **Install Dependencies**

   ```bash
   pip install -r requirements.txt
   ```

4. **Configure the Environment**
   Copy the example configuration and edit it with your device IPs:

   ```bash
   cp .env.example .env
   # Edit .env with your favorite editor
   ```

5. **Run the Application**
   ```bash
   python src/main.py
   ```

## 📂 Project Structure

- **`src/main.py`**: The entry point. Runs the main loop (~10s intervals) to orchestrate data fetching and processing.
- **`src/daily_data_processor.py`**: The "Brain" of the history logic. It takes raw 10s data point streams and resamples them into 10-minute average batches for the database.
- **`src/database.py`**: Manages the SQLite connection and schema (tables for daily summaries and intraday history).
- **`src/solar_data.py`** & **`src/shelly_power_data.py`**: Modules dedicated to communicating with the respective hardware APIs.
- **`src/power_solar_usage.py`**: logical unit that calculates derived values like "Self Usage" vs "Export".

## 📊 Database Schema

The local SQLite database (`solar_data.db`) contains:

- **`daily_solar`**: High-level daily stats (Total kWh, Peaks).
- **`daily_power_usage`**: Grid interaction stats (Import/Export/Self-use).
- **`intraday_history`**: The detailed 10-minute interval data for graphing daily curves.
